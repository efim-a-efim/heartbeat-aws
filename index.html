    AWS resource agents for Heartbeat

    Project maintained by efim-a-efim Hosted on GitHub Pages — Theme by mattgraham

    OCF resource agents and scripts for Heartbeat to interact with AWS.

    Originaly, it was developed as a patch for VyOS to live in HA configuration in AWS.

    Installation pre-requisties

    1. You must install Amazon AWS tools. aws utility must be in PATHfor user running heartbeat (e.g. root).
    2. You must launch your instance with AWS IAM role. This role must permit instance to list, edit, get properties of VPC route tables, IP addresses and Elastic IPs.

    AWS_IP agent

    Add Private IP to the active instance's interface.

    Parameters (in order of appearance):

    1. address - IP address to assign
    2. interface - network interface name (e.g. eth0, etc.)
    3. aws_role - IAM role name

    AWS_EIP agent

    Associate existing Elastic IP address with active instance's interface

    Parameters (in order of appearance):

    1. address - IP address to assign
    2. interface - network interface name (e.g. eth0, etc.)
    3. privateip - Private IP address. May be empty, then the first assigned address will be used.
    4. aws_role - IAM role name

    AWS_Route agent

    Change VPC route table to direct route through current instance's interface.

    Parameters (in order of appearance):

    1. table - Routing table ID
    2. destination - Route destination address/mask (e.g. 10.0.0.0/8)
    3. interface - network interface name (e.g. eth0, etc.)
    4. aws_role - IAM role name
